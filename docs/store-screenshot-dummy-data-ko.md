# 스토어 스크린샷용 더미 데이터 가이드 (KO)

최종 업데이트: 2026-03-11

이 문서는 Play 스토어 스크린샷 촬영용 더미 데이터 세트와 화면 구성 가이드를 정리합니다.

## 1. 가져오기 파일

사용 파일:

`docs/samples/store-screenshot-sample.csv`

앱에서 가져오기:

1. `옵션 > 파일`
2. `읽기: CSV/엑셀/PPF(PDF)`
3. `store-screenshot-sample.csv` 선택

이 CSV는 아래 목적을 위해 구성했습니다.

- 메인 화면에 수입/지출 요약 표시
- 거래 목록에 다양한 소비 유형 노출
- 자동 분류 화면에 식비/교통/쇼핑/통신/구독/주거/금융 계열 데이터 표시
- 예산 진행도 화면에 카테고리별 사용액 표시

## 2. 추천 예산 값

가져오기 후 `옵션 > 예산/분석` 에서 아래 값을 넣으면 화면이 가장 보기 좋습니다.

### 총 예산

`600000`

### 카테고리 예산

- `식비` → `120000`
- `교통` → `100000`
- `쇼핑` → `120000`
- `통신/구독` → `120000`
- `주거/공과금` → `120000`

이 조합이면 일부 카테고리는 적당히 차고, 일부는 여유가 남아서 진행 바가 보기 좋게 나옵니다.

## 3. 스크린샷별 권장 데이터 상태

### 1장 메인 대시보드

- CSV 가져오기 완료
- 최근 거래가 최소 4개 이상 보이도록 유지
- 수입/지출 요약이 모두 0이 아니어야 함

### 2장 파일 가져오기 화면

- `옵션 > 파일` 화면
- `읽기: CSV/엑셀/PPF(PDF)` 와 내보내기 버튼이 같이 보이게 캡처

### 3장 거래 목록 / 자동 분류 화면

- 거래 목록에 아래 항목이 같이 보이면 좋음
  - `급여 입금`
  - `카페 결제`
  - `지하철 충전`
  - `넷플릭스 구독`
  - `쿠팡 주문`

### 4장 예산 진행도 화면

- 총 예산 입력 완료
- 카테고리 예산 최소 4개 이상 저장
- 진행 바와 `남은 예산` 텍스트가 같이 보이게 캡처

### 5장 내보내기 / 마무리 화면

- `옵션 > 파일` 화면에서 `내보내기: 은행용 CSV`, `내보내기: 분석 CSV` 버튼이 같이 보이게 캡처
- 또는 `옵션 > 예산/분석` 의 월말 정산 카드가 함께 보이는 화면 사용

## 4. 추천 거래 노출 순서

목록 화면에서 아래 거래가 보이면 스토어 문구와 잘 맞습니다.

- `급여 입금`
- `카페 결제`
- `지하철 충전`
- `쿠팡 주문`
- `넷플릭스 구독`

## 5. 촬영 전 정리 팁

- 테스트용 이상한 상호명은 쓰지 않기
- 빈 상태 화면은 피하기
- 거래가 너무 많으면 최근 5~8개만 잘 보이도록 스크롤 위치 조정
- 폰 상태바 배터리/시간이 지나치게 지저분하면 정리 후 촬영
- 가능하면 같은 테마와 같은 언어 상태로 전 장면 촬영
